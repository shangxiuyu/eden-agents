# 功能设计工作流

## 工作流概述
基于福格行为模型，系统化地设计新功能和流程，确保用户能够自然完成目标行为。

## 第一阶段：需求理解

### 1.1 明确设计目标
询问并确认：
- 要设计什么功能？
- 为什么要做这个功能？
- 期望达成什么目标？
- 成功的标准是什么？

### 1.2 理解用户和场景
- 目标用户是谁？
- 用户的核心诉求是什么？
- 典型使用场景有哪些？
- 使用频率如何？

### 1.3 了解约束条件
- 技术限制
- 时间限制
- 资源限制
- 业务限制

## 第二阶段：定义目标行为

### 2.1 行为具体化
使用Who-What-When框架定义：
- Who: 哪类用户
- What: 具体做什么动作(可观察、可衡量)
- When: 在什么情境/时机下做

示例：
✅ "企业管理员在添加新成员时，设置其角色权限"
❌ "管理用户权限"(太模糊)

### 2.2 设计完整流程
拆解用户旅程的每个环节：
1. **发现入口**：用户如何知道这个功能？
2. **理解价值**：用户如何理解功能价值？
3. **启动行为**：用户如何开始使用？
4. **完成操作**：用户如何完成核心动作？
5. **获得反馈**：用户如何感知到成果？
6. **再次使用**：用户如何形成使用习惯？

### 2.3 设定衡量指标
- 使用率：多少用户使用？
- 完成率：多少用户完成？
- 效率：完成需要多久？
- 留存：多少用户重复使用？

## 第三阶段：MAT设计

### 3.1 动机(M)设计

**目标：让用户想要做这个行为**

#### 价值传递策略
- 功能价值：这个功能能帮用户做什么？
- 即时收益：用户能立即获得什么？
- 长期价值：持续使用有什么好处？

#### 动机强化手段
- **展示价值**：案例、数据、对比
- **社交证明**："1000+用户在使用"
- **个性化**："根据你的使用习惯推荐"
- **进度感**："完成度80%，加油！"
- **成就感**：徽章、里程碑、排行

#### 动机类型选择
根据场景选择合适的动机类型：
- 追求快乐：娱乐、社交、创造
- 避免痛苦：效率、安全、便捷
- 追求希望：成长、机会、愿景
- 避免恐惧：风险、损失、遗漏
- 追求认同：地位、影响、归属
- 避免排斥：合规、主流、安全

### 3.2 能力(A)设计

**目标：让用户能够轻松完成**

#### 简化六要素
- **Time**: 减少完成时间
  - 智能默认值
  - 批量操作
  - 自动化流程

- **Money**: 降低经济成本
  - 免费试用
  - 分期付款
  - 物超所值

- **Physical Effort**: 减少操作步骤
  - 一键完成
  - 拖拽操作
  - 快捷键

- **Brain Cycles**: 降低认知负担
  - 清晰标签
  - 视觉引导
  - 进度提示

- **Social Deviance**: 符合习惯
  - 遵循惯例
  - 渐进式改变
  - 熟悉的模式

- **Non-Routine**: 降低学习成本
  - 新手引导
  - 示例模板
  - 实时帮助

#### 简化设计原则
1. **减少步骤**：能合并就合并
2. **智能默认**：最常见的选择作为默认
3. **渐进式**：先简单后复杂
4. **容错性**：可撤销、可恢复
5. **清晰指引**：tooltips、examples、placeholders

#### 能力提升策略
- **新手引导**：首次使用的step-by-step
- **智能推荐**：基于上下文的建议
- **模板库**：常见场景的预设模板
- **快捷操作**：高频操作的快捷方式
- **帮助文档**：上下文敏感的帮助

### 3.3 触发(T)设计

**目标：在恰当时机提醒用户**

#### 触发类型选择
根据M和A的状态选择：

**Spark(火花型) - M低A高时**
- 目的：激发动机
- 示例："新功能上线！立即体验"
- 特点：强调价值和好处

**Facilitator(促进型) - M高A低时**
- 目的：降低门槛
- 示例："一键导入，30秒完成"
- 特点：强调简单易用

**Signal(信号型) - M高A高时**
- 目的：简单提醒
- 示例："您有3条待处理任务"
- 特点：清晰简洁

#### 触发时机设计
- **需求时刻**：用户需要时出现
- **能力时刻**：用户有能力时出现
- **动机时刻**：用户有意愿时出现
- **习惯时刻**：固定时间/地点

#### 触发方式设计
- **视觉触发**：通知、badge、高亮
- **文案触发**：标题、描述、CTA
- **环境触发**：位置、时间、事件
- **社交触发**：他人行为、消息

#### 触发原则
- **可见性**：用户能看到
- **相关性**：与当前场景相关
- **清晰性**：明确要做什么
- **时机性**：在合适的时候出现
- **适度性**：不要过度打扰

### 3.4 MAT平衡检查

在设计完成后评估：
- M维度评分：1-10分
- A维度评分：1-10分(���低越好，表示越简单)
- T维度评估：类型是否匹配，时机是否合适

在行为曲线上标注：
```
动机 ↑
     │     ╱
   高 │   ╱ [设计目标位置]
     │ ╱
   低 │─────────────→
     └───────────→ 能力
       难       易
```

确保设计在行为发生区内！

## 第四阶段：交互设计

### 4.1 信息架构
- 功能入口放在哪里？
- 信息如何组织？
- 层级关系如何？

### 4.2 界面布局
- 关键元素的位置
- 视觉层级
- 留白和节奏

### 4.3 交互细节
- 点击/悬停/拖拽
- 动画和过渡
- 加载和等待
- 错误处理

### 4.4 文案设计
- 功能名称
- 说明文案
- CTA按钮文案
- 提示和错误信息

## 第五阶段：实施规划

### 5.1 MVP定义
- 核心功能最小集
- 必须有的要素
- 可以延后的功能

### 5.2 分阶段计划
- Phase 1: 核心流程
- Phase 2: 体验优化
- Phase 3: 高级功能

### 5.3 资源评估
- 设计资源
- 开发资源
- 测试资源
- 时间估算

### 5.4 风险识别
- 技术风险
- 用户接受度风险
- 时间风险
- 依赖风险

## 第六阶段：验证与迭代

### 6.1 验证方法
- **原型测试**：可用性测试
- **A/B测试**：数据验证
- **灰度发布**：小范围试错
- **用户访谈**：深度反馈

### 6.2 监控指标
- 使用率：多少人在用
- 完成率：多少人完成
- 时长：完成需要多久
- 错误率：遇到问题的比例
- 留存：多久用一次

### 6.3 迭代策略
- 快速试错
- 数据驱动
- 持续优化
- 用户共创

## 输出格式

### 功能设计文档结构
1. **功能概述**
   - 背景和目标
   - 目标用户
   - 成功指标

2. **目标行为定义**
   - Who/What/When
   - 完整流程
   - 关键节点

3. **MAT设计**
   - 动机设计方案
   - 能力设计方案
   - 触发设计方案
   - 平衡性评估

4. **交互设计**
   - 信息架构
   - 界面设计
   - 交互细节
   - 文案设计

5. **实施计划**
   - MVP范围
   - 分阶段计划
   - 资源需求
   - 风险预警

6. **验证与迭代**
   - 验证方法
   - 监控指标
   - 迭代策略

✅ 目标行为定义清晰？
✅ MAT三要素设计完整？
✅ 在行为曲线的发生区内？
✅ 流程简洁高效？
✅ 触发时机恰当？
✅ 有即时反馈机制？
✅ 考虑了边界情况？
✅ 有数据监控计划？
✅ 符合伦理原则？
✅ 可实施且资源匹配？
