# 产品分析工作流

## 工作流概述
基于福格行为模型和系统分析思维，对现有产品进行全面诊断的标准流程。

## 第一阶段：理解背景

### 1.1 明确分析目标
询问并确认：
- 分析的具体目的是什么？
- 期望解决什么问题？
- 成功的标准是什么？

### 1.2 收集产品信息
- 产品类型(B端/C端)
- 目标用户群体
- 核心功能和价值主张
- 当前遇到的问题或挑战

### 1.3 获取数据基线
- 关键指标现状(转化率、留存率等)
- 用户反馈和痛点
- 竞品对比数据

## 第二阶段：识别目标行为

### 2.1 明确期望行为
定义清晰的目标行为：
- Who: 谁要做这个行为
- What: 具体做什么(可观察、可衡量)
- When: 在什么情境下做

示例：
✅ "新注册用户在首次登录后24小时内完成首个项目创建"
❌ "提升用户活跃度"(太模糊)

### 2.2 拆解行为路径
映射完整的用户旅程：
1. 认知阶段：用户知道这个功能吗？
2. 理解阶段：用户明白它的价值吗？
3. 意愿阶段：用户想要使用吗？
4. 行动阶段：用户开始操作了吗？
5. 完成阶段：用户成功完成了吗？
6. 重复阶段：用户会再次使用吗？

### 2.3 标记流失点
- 每个阶段的流失率
- 最严重的流失环节
- 流失的可能原因假设

## 第三阶段：MAT分析

### 3.1 动机(M)维度分析

**评估当前动机强度(1-10分)：**
- 用户对这个行为的内在驱动力
- 外部激励是否充足
- 价值感知是否清晰

**动机类型判断：**
- 追求快乐 vs 避免痛苦
- 追求希望 vs 避免恐惧
- 追求认同 vs 避免排斥

**动机障碍识别：**
- 价值不清晰
- 即时收益不明显
- 长期价值难感知
- 缺乏社交证明

### 3.2 能力(A)维度分析

**评估当前难度(1-10分，分数越高越难)：**

按六要素拆解：
- Time: 完成需要多长时间？
- Money: 有经济成本吗？
- Physical Effort: 需要多少操作步骤？
- Brain Cycles: 需要多少思考和决策？
- Social Deviance: 是否违背习惯或规范？
- Non-Routine: 是否需要学习新东西？

**能力障碍识别：**
- 步骤过多
- 操作复杂
- 信息不清晰
- 缺乏指引
- 容错性差
- 学习成本高

### 3.3 触发(T)维度分析

**当前触发评估：**
- 触发是否存在？
- 触发类型是否匹配？
  - Spark(激发动机): M低时使用
  - Facilitator(促进操作): A低时使用
  - Signal(简单提醒): M和A都高时使用
- 触发时机是否合适？
- 触发可见性如何？

**触发障碍识别：**
- 没有触发
- 触发时机不当(太早/太晚)
- 触发类型错配
- 触发不够显眼
- 触发与行为关联不清

### 3.4 综合判断

在行为曲线上标注当前位置：
```
动机 ↑
     │     ╱
   高 │   ╱ 行为发生区
     │ ╱ [当前位置?]
   低 │─────────────→
     └───────────→ 能力
       难       易
```

判断：
- 是否在行为发生区内？
- 最大的瓶颈在哪个维度？
- 哪个维度最容易优化？

## 第四阶段：诊断结论

### 4.1 问题总结
用MAT框架总结核心问题：
- M维度：动机方面的主要问题
- A维度：能力方面的主要问题
- T维度：触发方面的主要问题

### 4.2 根因分析
使用5WHY深挖：
- 为什么会出现这些问题？
- 根本原因是什么？
- 有哪些隐藏的制约因素？

### 4.3 影响评估
- 当前状态对业务的影响
- 不解决的后果
- 解决后的预期收益

## 第五阶段：优化建议

### 5.1 方案设计原则
- 优先解决最大瓶颈
- 简化优先(A维度成本最低)
- 全局平衡三要素
- 考虑实施可行性

### 5.2 具体优化建议

**针对M(动机)：**
- 如何强化价值感知
- 如何增强即时反馈
- 如何利用社交证明
- 如何设计激励机制

**针对A(能力)：**
- 如何减少操作步骤
- 如何降低认知负担
- 如何提供清晰指引
- 如何提升容错性

**针对T(触发)：**
- 如何优化触发时机
- 如何选择触发类型
- 如何提升触发可见性
- 如何强化触发关联

### 5.3 优先级排序
使用影响-成本矩阵：
- 高影响低成本 → 优先实施
- 高影响高成本 → 分阶段实施
- 低影响低成本 → 快速尝试
- 低影响高成本 → 暂缓

### 5.4 实施建议
- 分阶段方案
- 关键里程碑
- 资源需求
- 风险预警

## 第六阶段：监控计划

### 6.1 关键指标
- 北极星指标
- 过程指标
- 健康度指标

### 6.2 验证方法
- A/B测试设计
- 灰度发布计划
- 用户反馈收集

### 6.3 迭代机制
- 多久复盘一次
- 如何调整方案
- 何时扩大范围

## 输出格式

### 分析报告结构
1. **执行摘要**
   - 核心发现
   - 关键建议
   - 预期影响

2. **背景与目标**
   - 产品概况
   - 分析目的
   - 成功标准

3. **目标行为定义**
   - Who/What/When
   - 用户旅程
   - 流失分析

4. **MAT深度分析**
   - 动机分析
   - 能力分析
   - 触发分析
   - 综合诊断

5. **优化方案**
   - 核心建议
   - 优先级排序
   - 实施路径

6. **监控与迭代**
   - 关键指标
   - 验证方法
   - 迭代计划

## 质量检查清单

✅ 目标行为定义清晰且可衡量？
✅ 用户旅程完整且有数据支撑？
✅ M、A、T三维度分析深入？
✅ 根因分析到位？
✅ 优化建议具体可执行？
✅ 优先级判断合理？
✅ 有监控和迭代计划？
✅ 考虑了实施可行性？
