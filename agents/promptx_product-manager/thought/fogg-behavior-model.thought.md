# 福格行为模型思维

## 核心公式
B = MAT
- B (Behavior): 目标行为
- M (Motivation): 动机
- A (Ability): 能力
- T (Trigger): 触发

## 思维框架

### 1. 三要素必须同时具备
行为发生需要三要素在同一时刻交汇：
- 足够的动机驱动用户想做
- 足够的能力让用户能做
- 有效的触发提醒用户去做

缺少任何一个，行为都不会发生。

### 2. 动机维度(Motivation)

**核心动机类型：**
- 追求快乐 / 避免痛苦
- 追求希望 / 避免恐惧
- 追求认同 / 避免排斥

**分析要点：**
- 用户为什么要做这个行为？
- 是内在动机还是外在动机？
- 动机强度是否足够？
- 如何增强或激活动机？

### 3. 能力维度(Ability)

**简化的六要素：**
- Time (时间): 需要多久完成？
- Money (金钱): 需要付出多少成本？
- Physical Effort (体力): 需要多少身体努力？
- Brain Cycles (脑力): 需要多少思考？
- Social Deviance (社会偏离): 是否违背社会规范？
- Non-Routine (非常规性): 是否打破习惯？

**核心原则：**
能力 = 简单性。越简单，能力越强。

**设计策略：**
- 降低操作步骤
- 减少认知负担
- 提供清晰指引
- 利用已有习惯

### 4. 触发维度(Trigger)

**触发三类型：**
- Spark (火花型): 激发动机 - 用于动机不足时
- Facilitator (促进型): 简化操作 - 用于能力不足时
- Signal (信号型): 单纯提醒 - 用于动机和能力都具备时

**有效触发条件：**
- 时机：在用户动机和能力都具备时出现
- 可见：用户必须能注意到
- 关联：与目标行为明确关联

### 5. 行为曲线

```
动机 ↑
     │     ╱
   高 │   ╱ 行为发生区
     │ ╱
   低 │─────────────→
     └───────────→ 能力
       难       易
```

**核心洞察：**
- 动机高 + 难度低 → 行为最容易发生
- 动机低时，必须极度简化
- 能力低时，必须强化动机
- 触发必须在曲线之上才有效

## 实践应用

### 分析现有产品
1. 识别目标行为
2. 评估当前M、A、T各维度
3. 找出薄弱环节
4. 提出优化方案

### 设计新功能
1. 明确期望行为
2. 设计动机机制
3. 简化操作流程(提升能力)
4. 设计有效触发

### 优化转化漏斗
在每个流失点应用MAT分析：
- 动机是否足够？如何强化？
- 操作是否简单？如何简化？
- 触发是否有效？时机是否合适？

## 常见误区

1. **只关注动机**：疯狂激励但操作太复杂
2. **只关注能力**：功能很简单但用户没兴趣
3. **忽视触发时机**：提醒太早或太晚
4. **单一要素优化**：需要全局平衡三要素

## 决策原则

- 优先提升最薄弱的要素
- 简化永远是首选(能力维度成本最低)
- 动机难以改变,能力相对容易
- 触发要精准,不要打扰
- 用数据验证假设
